## このセットで必ず実行するテスト（省略禁止）

以下の 6 項目を**毎回すべて実行すること**:

1. R1 の `ospf_neighbors`（R2・R3 との 2 ネイバーが FULL であること）
2. R2 の `ospf_neighbors`（R1・R3 との 2 ネイバーが FULL であること）
3. R3 の `ospf_neighbors`（R1・R2 との 2 ネイバーが FULL であること）
4. R1 から `2.2.2.2` への `ping`（R2 Loopback 到達確認）
5. R1 から `3.3.3.3` への `ping`（R3 Loopback 到達確認）
6. R2 から `3.3.3.3` への `ping`（R3 Loopback 到達確認）

## このセット固有の失敗パターンと対策

### OSPF ネイバーが全台 0 の場合

- `network` ステートメントが `area 0` に漏れている可能性が高い
- `router-id` に Loopback0 アドレスを設定していない
- GigabitEthernet が `shutdown` 状態

正しい設定例:
```
router ospf 1
 router-id 1.1.1.1
 network 0.0.0.0 255.255.255.255 area 0
```

### OSPF ネイバーが一部のルーターで 1 つしか確立しない場合

- インターフェースの IP アドレスが要件と異なる（サブネットミスマッチ）
- 対向インターフェースが同じ /30 サブネットに属していない

### ping が通らない場合

- OSPF でルートが学習されていない → `route_table` テストで `1.1.1.1/32` 等を確認する
- Loopback0 の `network` 漏れ（`0.0.0.0 255.255.255.255` であれば包含されるはず）

## 障害シミュレーション時の期待動作

- R1–R2 リンク断: R1→R3→R2 の経路で `ping 2.2.2.2` が継続して成功すること
- R1–R3 リンク断: R1→R2→R3 の経路で `ping 3.3.3.3` が継続して成功すること
- R2–R3 リンク断: R2→R1→R3 の経路で `ping 3.3.3.3` が継続して成功すること

フルメッシュ構成のため、1 リンク断でも全 ping テストが PASS であること。
